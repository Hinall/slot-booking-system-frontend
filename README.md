# Slot Booking System — Frontend (React)

React (CRA) frontend for a lightweight scheduling system (a simplified Calendly).

- **User (host) side**: sign in, add availability, share booking link
- **Visitor side**: discover hosts (or open a shared link), view available dates/slots, book a time

---

## Project overview

You’re building a lightweight scheduling system where users can share their availability and visitors can book time with them

### What’s implemented

- **Authentication**: register → OTP verify → login (JWT access token + refresh cookie)
- **Host availability**: host adds date + start time + end time and sees the list in a dashboard accordion
- **Public booking**:
  - Visitors can **discover hosts** from Home → Discover (Users page)
  - Or open a **direct booking link** `/book/:userId(link Generated By host)`
  - Visitors select an **available date tab**, pick a **30‑minute slot**, and book
  - **Conflict-free booking**: already-booked slots are shown as **Booked** and cannot be selected

## Routes

- `/` — Home (visitor landing)
- `/signin` — Sign in (user)
- `/dashboard` — User dashboard (add availability + share booking link)
- `/users` — Visitor: list all users (hosts)
- `/book/:userId` — Visitor: booking page for a selected user

---

## Environment

Create a `.env` file:

```env
REACT_APP_API_URL=http://localhost:3000/api
```

Restart the dev server after changing `.env`.

---

## Run locally

```bash
npm install
npm start
```

The dev server runs on `http://localhost:3001` (or whichever port CRA selects if 3000 is already in use).
